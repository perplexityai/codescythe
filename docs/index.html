Config first

Make the source boundary explicit

Codescythe is designed for repositories where entry points and project files are known. The analyzer follows that graph rather than guessing through framework plugins. 

{
  "$schema": "./codescythe.schema.json",
  "entry": ["src/index.ts"],
  "project": ["src/**/*.{ts,tsx}"],
  "testFilePatterns": ["**/*.test.*"],
  "ignore": ["src/generated/**"]
}

Analyze before editing

JSON output includes unused files, unused exports, unresolved imports, matched config, and verbose diagnostics for reviewable cleanup runs.

Apply supported fixes

The fix pass removes unused project files and export declarations, then reports changed files and skipped export edits when resolver uncertainty makes a change risky.

Check config risk

Doctor mode flags broad ignores, unresolved imports, empty entry globs, and generated-source patterns that overlap checked source.

Package and binary releases

Run it from npm or GitHub releases

The public npm package loads the matching native Node-API package for supported platforms. Standalone CLI binaries are published for macOS ARM64, Linux AMD64, and Linux ARM64. 

npm install -D codescythe
npx codescythe --json --config codescythe.jsonc
npx codescythe doctor --config codescythe.jsonc
